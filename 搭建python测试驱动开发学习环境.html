搭建Python驱动测试开发学习环境

本来打算在Debian下搭建测试驱动开发学习环境，不过不知道是lxde不给力还是怎么，geckodriver始终不能正常工作，于是回到了Deepin的怀抱……

环境简介

  • 操作系统 Deepin 15.4.1
  • Python版本 Python 3.4
  • 包管理器 pip
  • Django版本 Django 1.7
  • 其他工具 Selenium
  • 浏览器 Firefox 52

部署过程

  1. 安装操作系统
  2. 下载系统镜像文件
  3. 在Windows下解压文件，取出其中安装盘制作工具
  4. 插入U盘制作安装盘
  5. 针对目标机器安装系统
  6. 安装Python 3.4
  7. 操作系统自带2.7和3.5版本的Python，通过apt-get安装Python3.4
  8. 安装pip（Python3.5）
  9. 将系统下/usr/bin中的python软连接从python2改为python3
  10. 通过apt-get安装python3-pip
  11. 通过pip安装virtualenv和virtualenvwrapper
  12. 创建指定Python3.4为解释器的虚拟环境
  13. 虚拟环境内，通过pip安装Django1.7和最新版Selenium
  14. 下载最新版geckodriver，解压至上述虚拟环境的python文件所在位置
  15. 书写第一个测试程序，正常报错，环境搭建完毕

经验教训

唯一的经验教训就是不要在不熟悉的发行版上学习，容易踩到深坑里
